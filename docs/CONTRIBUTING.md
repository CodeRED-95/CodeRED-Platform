# Contribuir

## Cómo colaborar

- Crear ramas cortas por cambio.
- Abrir PR con alcance claro.

## Estándares

- Español en documentación.
- Código consistente con el dominio.

## Commits

- Mensajes descriptivos.
- Un objetivo por commit.

## Branches

- `feature/...`
- `fix/...`
- `docs/...`

## Pull Requests

- Explicar el cambio.
- Indicar pruebas realizadas.
- Adjuntar capturas si cambia UI.

## Revisión de código

- Verificar seguridad.
- Verificar migraciones.
- Verificar que no se rompa el módulo Agencies.
